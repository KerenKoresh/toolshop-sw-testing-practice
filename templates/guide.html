What ToolShop does

ToolShop is a small full-stack demo store. It pairs a clean catalog UI with a fully public REST API, so the same products you browse here can also be created, searched, updated and deleted programmatically. It makes a realistic target for API testing, automation practice and demos.

How ownership works

There is no login. Reading and searching are open to everyone. When you create a product through the API, the response includes a one-time edit_token, a random secret. Only a SHA-256 hash of it is stored on the server. To later update or delete that product you must send the same token in the X-Edit-Token request header. Products in the seeded baseline catalog have no token and are therefore read-only.

The API, request by request

Base URL: /api. All responses are JSON. Below is exactly what each request does.

GET /api/products No auth

Returns the full catalog. Supports two optional query parameters: ?search=<text> for a partial, case-insensitive match on the product name, or ?id=<number> for an exact ID lookup (returns a list of 0 or 1 items).

curl "https://kerens-software-testing-practice.onrender.com/api/products?search=plier"
GET /api/products/{id} No auth

Returns a single product by its ID, or 404 if it does not exist.

curl "https://kerens-software-testing-practice.onrender.com/api/products/3"
POST /api/products No auth

Creates a product. name is required; description, price, category and in_stock are optional. The response includes the created product and a one-time edit_token (save it, it is the only way to modify the product later).

curl -X POST https://kerens-software-testing-practice.onrender.com/api/products \
  -H "Content-Type: application/json" \
  -d '{"name":"Rubber Mallet","price":9.9,"category":"Hammer"}'
PUT /api/products/{id} X-Edit-Token

Updates a product. Any omitted field keeps its current value. Requires the product's edit_token in the X-Edit-Token header. Returns 401 if the header is missing, 403 if the token is wrong or the product is a read-only baseline item, and 404 if it does not exist.

curl -X PUT https://kerens-software-testing-practice.onrender.com/api/products/13 \
  -H "Content-Type: application/json" \
  -H "X-Edit-Token: <your-token>" \
  -d '{"price":19.99,"in_stock":false}'
DELETE /api/products/{id} X-Edit-Token

Deletes a product. Requires the matching edit_token. Same error codes as update (401 / 403 / 404).

curl -X DELETE https://kerens-software-testing-practice.onrender.com/api/products/13 \
  -H "X-Edit-Token: <your-token>"
GET /api/health No auth

Lightweight health check. Returns {"status":"ok","db":"..."} including the active database engine (sqlite or postgresql).

curl "https://kerens-software-testing-practice.onrender.com/api/health"

Data retention

To keep the database small, products created through the API are temporary. A scheduled job runs once a day at midnight (Israel time) and removes every product created via the API, leaving only the built-in baseline catalog. If you create products for testing, expect them to be cleared at the next midnight.
